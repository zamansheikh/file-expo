## Correct IP detection & changelog viewer

- **Fixed**: the installer sometimes printed an internal Docker bridge address (e.g. `172.22.0.1`) instead of your server's real public IP. IP detection now forces IPv4, tries several services, falls back to the outbound route address, and skips private/Docker ranges
- **Fixed**: terminal spinner no longer breaks across lines during package installs
- **New: Changelog viewer** — click the version chip in the topbar to see what changed in every version, including what's coming in an available update before you install it
- Server startup log no longer lists Docker/bridge interface URLs
