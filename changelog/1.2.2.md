## Cache fix — Docker & Shares tabs showing an error

- **Fixed**: after updating (or re-installing), browsers could keep an old cached `app.js` alongside the new page, which broke the Docker and Shares tabs with a "TABS[id] is not a function" error
- Asset URLs are now version-stamped (`app.js?v=1.2.2`) and served with `no-cache`, so every update always loads matching files
- If a version mismatch is ever detected, the page auto-reloads once to fix itself
- Tools tabs now show a friendly "hard-refresh" hint instead of a raw error if they're ever out of sync
